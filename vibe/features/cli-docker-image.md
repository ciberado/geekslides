# CLI Docker Image

## Overview

The CLI Docker image (`ciberado/geekslides-cli`) packages the `@geekslides/cli` tool into
a portable Docker image. Running the image with no arguments outputs a self-installing
wrapper script; running it with arguments executes the CLI directly.

## Quick Start

```sh
# 1. Download the wrapper script (stdout is piped → emits script)
docker run --rm ciberado/geekslides-cli > geekslides
chmod +x geekslides

# Or just run it to see help (stdout is a terminal → shows help)
docker run --rm ciberado/geekslides-cli

# 2. Use it — all CLI commands work
./geekslides dev                          # start dev server on port 3000
./geekslides build --config config.json   # build production bundle
./geekslides create --title "My Talk"     # scaffold a new deck
./geekslides pdf --config config.json     # export PDF (requires :chromium tag)
```

## Image Variants

Two image tags are published from `docker/Dockerfile.cli`:

| Tag | Base | Size | PDF support |
|-----|------|------|-------------|
| `ciberado/geekslides-cli:latest` | `node:22-alpine` | ~200 MB | No — Chromium not included |
| `ciberado/geekslides-cli:chromium` | `node:22-bookworm-slim` | ~800 MB | Yes — Playwright + Chromium pre-installed |

Both images are built from the same multi-target Dockerfile:

```sh
# Slim (default)
docker build --target slim -t ciberado/geekslides-cli:latest -f docker/Dockerfile.cli .

# Chromium (PDF export)
docker build --target chromium -t ciberado/geekslides-cli:chromium -f docker/Dockerfile.cli .
```

## Architecture

### Entrypoint Modes

`docker/cli-entrypoint.sh` implements three modes based on arguments and stdout:

```
docker run ciberado/geekslides-cli            → stdout is TTY: prints help message
docker run ciberado/geekslides-cli > file      → stdout is piped: prints wrapper script
docker run ciberado/geekslides-cli dev ...     → runs CLI command directly
```

The TTY detection uses `[ -t 1 ]` (test if file descriptor 1/stdout is a terminal).
When no arguments are given and stdout is redirected or captured via `$(...)`, the
entrypoint emits the wrapper script. When stdout is an interactive terminal, it shows
a help message explaining how to install.

### Generated Wrapper Script

The wrapper script generated by the entrypoint:

1. Detects the CLI command (`dev`, `build`, `pdf`, `create`)
2. Mounts the deck directory (`$GEEKSLIDES_DIR` or `$PWD`) at `/deck` inside the container
3. Adds `-p` port mapping for the `dev` command (default: 3000)
4. Passes `--host 0.0.0.0` for `dev` to bind on all interfaces inside the container
5. Uses `-it` when stdin is a terminal, `--rm` always

### Port Mapping (dev command)

The `dev` command starts Vite (default 5173) and y-websocket (default 1234) internally.
The wrapper script maps the external port (`GEEKSLIDES_PORT`, default 3000) to the
internal Vite port, and passes `--port` to match:

```
Host :3000 → Container :3000 (Vite)
Container :1234 (y-websocket, internal only)
```

WebSocket sync is proxied through Vite at `/ws*`, so only one port needs exposing.

### Volume Mounts

The wrapper mounts the deck directory at `/deck` inside the container:

```sh
-v "${GEEKSLIDES_DIR:-$PWD}":/deck
```

The CLI runs with `/deck` as its working directory. Relative paths in `config.json`
(content, styles, images) resolve against `/deck`.

## Environment Variables

| Variable | Default | Where | Description |
|----------|---------|-------|-------------|
| `GEEKSLIDES_IMAGE` | `ciberado/geekslides-cli:latest` | Image | Baked into the wrapper script |
| `GEEKSLIDES_PORT` | `3000` | Image | Default port baked into the wrapper script |
| `GEEKSLIDES_DIR` | `$PWD` | Wrapper | Deck directory to mount (runtime override) |
| `GEEKSLIDES_PORT` | `3000` | Wrapper | Port for dev server (runtime override) |

## Dockerfile Structure

```
docker/Dockerfile.cli
├── Stage: builder (node:22-alpine)
│   ├── npm ci
│   ├── Build @geekslides/engine
│   ├── Build @geekslides/server
│   └── Build @geekslides/cli (esbuild → dist/index.cjs)
│
├── Target: slim (node:22-alpine)
│   ├── Copy CLI binary + app shell + engine source
│   ├── Install vite + sharp (runtime externals)
│   └── Entrypoint: cli-entrypoint.sh
│
└── Target: chromium (node:22-bookworm-slim)
    ├── Copy CLI binary + app shell + engine source
    ├── Install vite + sharp + playwright (runtime externals)
    ├── Install Chromium via Playwright
    └── Entrypoint: cli-entrypoint.sh
```

## Testing

Unit tests in `packages/cli/tests/unit/docker-wrapper.test.ts` validate:

- The entrypoint outputs a valid shell script when invoked with no arguments
- The generated script references the correct Docker image name
- The `dev` command gets port mapping added
- Environment variable overrides work (`GEEKSLIDES_DIR`, `GEEKSLIDES_PORT`)

Dockerfile builds are validated in CI as part of the Docker build matrix.

## Relationship to Server Image

The **server image** (`docker/Dockerfile`) bundles the SPA + Caddy + y-websocket into a
production deployment container. The **CLI image** (`docker/Dockerfile.cli`) bundles the
development toolchain for authoring presentations locally.

| Concern | Server image | CLI image |
|---------|-------------|-----------|
| Purpose | Serve presentations to audiences | Author and build presentations |
| Base | Alpine + Caddy | Alpine (slim) or Bookworm (chromium) |
| Ports | 80, 443 | 3000 (dev only) |
| Volumes | Content (read-only) | Deck directory (read-write) |
| Commands | Caddy + yjs-server | dev, build, pdf, create |
