# Slide HTML Reference

This document describes the **real HTML** produced by `SlideParser.parse()` for each common slide pattern. It is intended as a reference for writing CSS and for understanding how markdown content maps to DOM elements.

> **Source of truth:** All examples below were generated by running
> `packages/engine/tests/unit/html-output-capture.test.ts`. Re-run that
> test whenever the parser changes to verify this document is still current.

---

## How the HTML fits into the DOM

Each parsed slide produces a `SlideData` object. The `Slideshow` component creates a `<geek-slide>` custom element for every slide and calls `loadContent()`, which builds a Shadow DOM structure:

```
<geek-slide data-id="<id>" class="[user-classes…]" [active]>
  #shadow-root
    <style>                  ← base styles + scoped per-slide rawCss
    <section class="content [user-classes…]" id="<id>">
      ← slide.html injected here (innerHTML)
```

Parser output fields:

| Field             | Description |
|-------------------|-------------|
| `html`            | Main slide content; injected as `innerHTML` of `<section class="content">` |
| `notesHtml`       | Speaker notes (`::: Notes`); never rendered inside the slide itself |
| `detailsHtml`     | Book-mode content (`::: Details`); **also** kept inside `html` as `.gs-details` |
| `rawCss`          | CSS extracted from `<style>` blocks; scoped to the slide id |
| `classes`         | Applied to both `<geek-slide>` and `<section class="content">` |
| `id`              | `data-id` on `<geek-slide>`, `id` on `<section>` |
| `partialCount`    | Number of elements with `partial=""` attribute |
| `backgroundImage` | Applied as `style.backgroundImage` on `<geek-slide>` |
| `backgroundColor` | Applied as `style.backgroundColor` on `<geek-slide>` |

## Partials (progressive reveal)

Partial reveal marks elements with `partial=""` attribute. The `Slide` component then adds `.gs-partial` and `.gs-visible` classes during navigation.

### H1 + list with explicit `[partial]` markers

**Markdown**
```markdown
# Title

- First item [partial]
- Second item [partial]
- Third item [partial]
```

**HTML** (`slide.html` — as injected into the shadow DOM)
```html
<h1>Title</h1>
<ul>
  <li partial="">First item</li>
  <li partial="">Second item</li>
  <li partial="">Third item</li>
</ul>
```

`partialCount: 3`

> The `[partial]` marker text is stripped from the content; only the attribute
> remains. The `Slide` component converts `partial=""` → `.gs-partial`, and
> reveals them one-by-one as `.gs-visible`.

---

### H1 + list with `.mod-partial` class (auto-reveal)

**Markdown**
```markdown
[](.mod-partial#auto-partial)

# Title

- First item
- Second item
- Third item
```

**HTML**
```html
<h1>Title</h1>
<ul>
  <li partial="">First item</li>
  <li partial="">Second item</li>
  <li partial="">Third item</li>
</ul>
```

`partialCount: 3`, `classes: [partial]`, `id: auto-partial`

> Adding the `.mod-partial` class to the slide separator automatically marks every
> `<li>` (and `<tr>`) as partial — no need for per-item `[partial]` markers.

## Speaker Notes and Details

### H1 + speaker notes

**Markdown**
```markdown
# Title

Slide content here.

::: Notes
These are **speaker notes** that only appear in speaker view.
They can contain any markdown.
:::
```

**`slide.html`** (main slide — notes are absent)
```html
<h1>Title</h1>
<p>Slide content here.</p>
```

**`slide.notesHtml`** (rendered separately for speaker view)
```html
<p>These are <strong>speaker notes</strong> that only appear in speaker view.
They can contain any markdown.</p>
```

> Notes are completely removed from the main `html` field. They are only
> accessible through `SlideData.notesHtml` and rendered inside `<geek-speaker-view>`.

---

### H1 + details block

**Markdown**
```markdown
# Title

Slide content here.

::: Details
These are **details** that appear in book/export mode.
:::
```

**`slide.html`** (details remain present, wrapped in `.gs-details`)
```html
<h1>Title</h1>
<p>Slide content here.</p>
<div class="gs-details">
  <p>These are <strong>details</strong> that appear in book/export mode.</p>
</div>
```

**`slide.detailsHtml`** (also extracted for use by the print renderer)
```html
<p>These are <strong>details</strong> that appear in book/export mode.</p>
```

> Unlike notes, details **stay in the main `html`** wrapped in `.gs-details`.
> Presentation CSS can hide `.gs-details` while the print renderer reads
> `detailsHtml` directly for book-format PDF exports.

---

### H1 + notes + details (combined)

**Markdown**
```markdown
# Title

Slide content here.

::: Notes
Speaker notes text.
:::

::: Details
Book-mode details text.
:::
```

**`slide.html`**
```html
<h1>Title</h1>
<p>Slide content here.</p>
<div class="gs-details">
  <p>Book-mode details text.</p>
</div>
```

**`slide.notesHtml`**
```html
<p>Speaker notes text.</p>
```

**`slide.detailsHtml`**
```html
<p>Book-mode details text.</p>
```

## Slide attributes (separator syntax)

Slide attributes are set with the empty-link separator: `[](.class1.class2#id,bgurl(url),bgcolor(color))`.

### Custom classes and id

**Markdown**
```markdown
[](.hero.dark#my-slide)

# Title
```

**HTML**
```html
<h1>Title</h1>
```

`classes: [hero, dark]`, `id: my-slide`

> Classes are applied to **both** `<geek-slide>` (the host) and
> `<section class="content hero dark">` (inside the shadow DOM). Background
> and color attributes are set as inline styles on `<geek-slide>`.

---

### Background image

**Markdown**
```markdown
[](.mod-coverbg#bg-slide,bgurl(hero.jpg))

# Title on Background
```

**HTML**
```html
<h1>Title on Background</h1>
```

`classes: [mod-coverbg]`, `id: bg-slide`, `backgroundImage: hero.jpg`

The `backgroundImage` value is applied as `style="background-image: url(hero.jpg); background-size: cover; background-position: center;"` on the `<geek-slide>` element.

---

### Background color

**Markdown**
```markdown
[](.#color-slide,bgcolor(#1e1e2e))

# Dark Slide
```

**HTML**
```html
<h1>Dark Slide</h1>
```

`id: color-slide`, `backgroundColor: #1e1e2e`

Applied as `style="background-color: #1e1e2e;"` on `<geek-slide>`.

## Per-slide `<style>` blocks

**Markdown**
```markdown
# Styled Slide

<style>
h1 { color: hotpink; }
</style>

Content below styled heading.
```

**`slide.html`** (`<style>` tag is removed from the HTML)
```html
<h1>Styled Slide</h1>
<p>Content below styled heading.</p>
```

**`slide.rawCss`**
```css
h1 { color: hotpink; }
```

The parser extracts `<style>` blocks from the HTML and stores the CSS in `rawCss`. The `Slide` component then scopes the CSS to the slide's `id` via `StyleScoper` and injects it into the shadow DOM `<style>` element. This prevents leakage to other slides.

## Multiple lists

### H1 + two unordered lists (blank-line separated)

**Markdown**
```markdown
# Title

- List one, item A
- List one, item B

- List two, item A
- List two, item B
```

**HTML**
```html
<h1>Title</h1>
<ul>
  <li>
    <p>List one, item A</p>
  </li>
  <li>
    <p>List one, item B</p>
  </li>
  <li>
    <p>List two, item A</p>
  </li>
  <li>
    <p>List two, item B</p>
  </li>
</ul>
```

> **Important:** When two consecutive `- …` lists are separated by a blank line,
> markdown-it treats the combined sequence as a single **loose list** and wraps
> each item's content in `<p>`. The result is **one `<ul>`**, not two. This
> affects line-height and spacing in CSS. To get two separate lists you must
> place a non-list element (e.g. a paragraph, `<hr>`, or an `<h4>`) between them.

---

### H1 + unordered list + ordered list

**Markdown**
```markdown
# Title

- Bullet item A
- Bullet item B

1. Step one
2. Step two
```

**HTML**
```html
<h1>Title</h1>
<ul>
  <li>Bullet item A</li>
  <li>Bullet item B</li>
</ul>
<ol>
  <li>Step one</li>
  <li>Step two</li>
</ol>
```

Different list types (`<ul>` + `<ol>`) are **always** emitted as separate elements regardless of blank lines between them.

## Bold/italic first words in list items

When a list item begins with `**bold**` or `*italic*`, the inline element appears directly as the first child of `<li>` — there is no wrapping `<p>`. The rest of the text follows as a text node.

### H1 + list with bold first words

**Markdown**
```markdown
# Title

- **Key term one**: explanation of the first term.
- **Key term two**: explanation of the second term.
- **Key term three**: explanation of the third term.
```

**HTML**
```html
<h1>Title</h1>
<ul>
  <li><strong>Key term one</strong>: explanation of the first term.</li>
  <li><strong>Key term two</strong>: explanation of the second term.</li>
  <li><strong>Key term three</strong>: explanation of the third term.</li>
</ul>
```

> CSS tip: `li > strong:first-child` targets the bold label without wrapping it
> in an extra element.

---

### H1 + list with italic first words

**Markdown**
```markdown
# Title

- *First concept*: explanation of first concept.
- *Second concept*: explanation of second concept.
- *Third concept*: explanation of third concept.
```

**HTML**
```html
<h1>Title</h1>
<ul>
  <li><em>First concept</em>: explanation of first concept.</li>
  <li><em>Second concept</em>: explanation of second concept.</li>
  <li><em>Third concept</em>: explanation of third concept.</li>
</ul>
```

### H1 + ordered list with bold first words

**Markdown**
```markdown
# Title

1. **Install**: run `npm install`.

3. **Run**: execute `npm run dev`.
```

**HTML**
```html
<h1>Title</h1>
<ol>
  <li><strong>Install</strong>: run <code>npm install</code>.</li>
  <li><strong>Configure</strong>: edit config.json.</li>
  <li><strong>Run</strong>: execute <code>npm run dev</code>.</li>
</ol>
```

## Summary of generated elements

| Markdown construct | HTML output |
|--------------------|-------------|
| `# …` | `<h1>` |
| `## …` | `<h2>` |
| `### …` | `<h3>` |
| `#### …` | `<h4>` (rendered as regular flow, no special treatment) |
| `**text**` | `<strong>` |
| `*text*` | `<em>` |
| `` `code` `` | `<code>` |
| `- item` | `<ul><li>` |
| `1. item` | `<ol><li>` |
| Two `- …` lists, blank-line separated | **Merged** into one `<ul>` with `<li><p>` (loose list) |
| `- …` + `1. …` (different types) | Two separate `<ul>` + `<ol>` elements |
| Two `1. …` lists separated by non-list block | Two separate `<ol>` elements (tight, no `<p>` wrapping) |
| Nested list | `<li>` contains child `<ul>`/`<ol>` |
| `- **bold**: text` | `<li><strong>…</strong> text` — no wrapping `<p>` |
| `- *italic*: text` | `<li><em>…</em> text` — no wrapping `<p>` |
| `![alt](url)` (standalone) | `<div class="block-image"><img>` |
| GFM table | `<table><thead><tbody><tr><th>/<td>` |
| ` ``` lang ``` ` | `<pre><code class="language-lang">` |
| `> quote` | `<blockquote><p>` |
| `::: Notes … :::` | Removed from `html`; available in `notesHtml` |
| `::: Details … :::` | Kept in `html` as `<div class="gs-details">`; also in `detailsHtml` |
| `[partial]` marker | Strips text; sets `partial=""` on containing `<li>`/`<tr>`/`<p>` |
| `.mod-partial` slide class | Auto-adds `partial=""` to every `<li>` / `<tr>` |
| `<style>` block | Removed from `html`; stored in `rawCss` (scoped by `StyleScoper`) |
| Raw HTML | Passed through as-is |
