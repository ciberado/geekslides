# Use the Docker CLI

Run every GeekSlides command through Docker — no Node.js installation required. A single `docker run` produces a lightweight wrapper script that handles volume mounts, port mapping, and container lifecycle for you.

## Install the wrapper script

Pull the image and generate the script:

```bash
docker run --rm ciberado/geekslides-cli > geekslides
chmod +x geekslides
```

Move it somewhere on your `PATH` for global access:

```bash
sudo mv geekslides /usr/local/bin/
```

> **Tip:** Running the image without redirection shows a help message with usage instructions:
>
> ```bash
> docker run --rm ciberado/geekslides-cli
> ```

## Create a presentation

```bash
mkdir my-talks && cd my-talks
geekslides create --title "My First Deck"
cd my-first-deck
```

This scaffolds `config.json`, `README.md`, `css/local.css`, and an `images/` directory inside the current folder.

## Check the version

```bash
geekslides --version
```

## Start the dev server

```bash
geekslides dev
```

Open `http://localhost:3000` in your browser. The wrapper script:

1. Mounts your current directory into the container at `/deck`
2. Maps port 3000 on the host to the Vite dev server inside the container
3. Binds Vite on `0.0.0.0` so the port mapping works

Edit `README.md` in your editor — changes appear instantly via HMR.

### Custom port

```bash
GEEKSLIDES_PORT=8080 geekslides dev
```

### Custom deck directory

```bash
GEEKSLIDES_DIR=~/presentations/my-talk geekslides dev
```

## Build a production bundle

```bash
geekslides build --config config.json
```

The output lands in `dist/` inside your deck directory.

## Export to PDF

PDF export requires Chromium, which is included in the `:chromium` image tag. Re-generate the wrapper with that tag:

```bash
docker run --rm ciberado/geekslides-cli:chromium > geekslides
chmod +x geekslides
```

Then export:

```bash
geekslides pdf --config config.json
geekslides pdf --config config.json --format slides-notes
geekslides pdf --config config.json --format book
```

> **Tip:** The `:chromium` image is larger (~800 MB vs ~200 MB). Use the default `:latest` tag if you don't need PDF export.

## Image variants

| Tag | Base | Size | PDF support |
|-----|------|------|-------------|
| `ciberado/geekslides-cli:latest` | Alpine | ~200 MB | No |
| `ciberado/geekslides-cli:chromium` | Debian Bookworm | ~800 MB | Yes |

## Build the images locally

From the repository root:

```bash
# Slim (default)
docker build --target slim -t ciberado/geekslides-cli:latest -f docker/Dockerfile.cli .

# With Chromium for PDF export
docker build --target chromium -t ciberado/geekslides-cli:chromium -f docker/Dockerfile.cli .
```

## Environment variables

| Variable | Default | Description |
|----------|---------|-------------|
| `GEEKSLIDES_DIR` | Current directory (`$PWD`) | Deck directory to mount into the container |
| `GEEKSLIDES_PORT` | `3000` | Port for the `dev` command |

## How it works

The entrypoint detects how stdout is connected:

- **Terminal (interactive)** — shows a help message with usage instructions
- **Piped/redirected** (`> file` or `$(...)`) — outputs the wrapper script
- **With arguments** — runs the CLI command directly inside the container

The wrapper script generated by the entrypoint is a thin shell script that:

1. Detects whether stdin is a terminal (adds `-it` if so)
2. Always adds `--rm` to clean up the container after exit
3. Mounts `$GEEKSLIDES_DIR` (or `$PWD`) at `/deck` inside the container
4. For the `dev` command: adds `-p` port mapping and `--host 0.0.0.0`
5. Forwards all arguments to the CLI binary inside the container

You can inspect it at any time:

```bash
cat $(which geekslides)
```

## Troubleshooting

| Problem | Solution |
|---------|----------|
| `permission denied` when running `./geekslides` | Run `chmod +x geekslides` |
| Port already in use | Set a different port: `GEEKSLIDES_PORT=9000 geekslides dev` |
| Files not visible in the container | Check that `$PWD` or `$GEEKSLIDES_DIR` points to your deck |
| PDF export fails with "browser not found" | Use the `:chromium` tag to regenerate the wrapper |
| Docker not found | Install Docker: [docs.docker.com/get-docker](https://docs.docker.com/get-docker/) |
